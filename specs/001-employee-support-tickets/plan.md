# Implementation Plan: Employee Support Ticket Management

**Branch**: `001-employee-support-tickets` | **Date**: 2026-03-06 | **Spec**: `specs/001-employee-support-tickets/spec.md`  
**Input**: Feature specification from `specs/001-employee-support-tickets/spec.md`

## Summary

Enable employees to view their assigned support tickets, read and reply to ticket thread conversations, and close tickets by setting status to `RESOLVED`. The feature requires: one new backend endpoint (`PATCH /api/ticket/{ticketId}/close`), a bug-fix to the thread-create authorization guard, a one-line client enum correction, and a "Close Ticket" UI button in the existing modal gated on employee role and ticket status.

## Technical Context

**Language/Version**: Java 21 (server), TypeScript / Angular 21 (client)  
**Primary Dependencies**:
- Server: Spring Boot 3.x, Spring Security 6, Spring Data JPA, Lombok, PostgreSQL driver, Flyway
- Client: Angular 21, Angular Material 19, RxJS, `@angular/forms`, `HttpClient`

**Storage**: PostgreSQL (local: `localhost:5432/chinook`; production: AWS RDS via `application-rds.properties`)  
**Testing**: JUnit 5 + Spring Boot Test (server); Vitest 4.x with jsdom (client)  
**Target Platform**: Linux server (Spring Boot API) + browser SPA (Angular)  
**Project Type**: Web application — REST API backend + Angular SPA frontend  
**Performance Goals**: Ticket list and thread load within 2 seconds for typical data sets (≤50 tickets, ≤100 thread messages)  
**Constraints**: JWT stateless auth; secrets via env vars; Flyway-only schema changes; `EMPLOYEE` role enforcement on all write/close operations  
**Scale/Scope**: Single-tenant; one employee ↔ many customers; one customer ↔ one ticket per support issue
